# Story SP-10.5: 减重日历 (Weight Calendar)

## 📋 用户故事

**作为** 用户  
**我想要** 一个可视化日历，展示我在哪些日子有记录体重等健康数据  
**以便** 我能直观地看到我的记录习惯和健康数据趋势

---

## 🎯 验收标准

### 日历展示
- [ ] **月份视图**: 显示当前月份的日历网格，带年月导航
- [ ] **记录标识**: 用颜色或图标突出显示有记录的日期
- [ ] **数据预览**: 日期格子上简要显示相关健康数据（如体重数值）
- [ ] **日历导航**: 支持左右箭头切换月份

### 交互功能
- [ ] **日期点击**: 点击日期可跳转到该天的具体数据或报告
- [ ] **当前日期高亮**: 今天的日期应该突出显示
- [ ] **月份切换**: 可以平滑切换不同月份

### 数据来源
- [ ] **体重记录**: 醍取用户体重历史数据并显示在对应日期
- [ ] **打卡标识**: 显示有习惯打卡、运动或饮食记录的日期
- [ ] **数据聚合**: 综合显示某日的多项健康数据

### 界面设计
- [ ] **视觉清晰**: 日期格子尺寸适中，确保信息清晰可读
- [ ] **一致性**: 与应用的整体设计风格和色彩搭配保持一致
- [ ] **响应式**: 在不同屏幕尺寸下都有良好的显示效果

## 📐 技术规格

### 文件位置

**新建文件:**
- `frontend/src/components/WeightCalendarView.tsx` - 减重日历组件

**整合到页面:**
- `frontend/src/pages/HealthReports.tsx` - 将日历组件集成到报告页面

### 组件规范

```javascript
// WeightCalendarView.tsx
interface Props {
  onDayClick: (dateData) => void; // 日期点击回调
}

// 日期数据接口
interface DayData {
  date: number;        // 日期（1-31）
  dateStr: string;     // ISO日期字符串 "YYYY-MM-DD"
  weight?: number;     // 体重数值
  hasOtherData: boolean;  // 是否有其他健康数据
  habitCompletion: boolean; // 习惯打卡情况
}
```

## 🛠️ 实施建议

### 数据集成策略
1. **前端Mock**: 先创建前端组件和UI界面，使用_mock数据
2. **后端API**: 实现支持日历所需日期汇总的API端点
3. **数据绑定**: 将真实API数据绑定到日历组件

### API 端点
- 可能需要 `GET /reports/calendar-data?month=YYYY-MM` 来获取当月健康数据摘要

### 性能优化
1. **数据分页**: 防止获取过多历史数据影响性能
2. **局部更新**: 仅刷新需要变化的数据
3. **缓存机制**: 缓冲访问过的月份数据

---

## 🚀 开发优先级

1. **基础日历**: 实现静态月份日历网格
2. **数据可视化**: 展示体重数据在日历中的标识
3. **交互功能**: 添加日期点击事件和导航功能
4. **完整集成**: 连接真实后端数据
5. **UX优化**: 根据用户反馈调整UI和交互细节
