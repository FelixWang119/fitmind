# Story SP-10.4: 健康报告页面 (Health Report Page)

## 📋 用户故事

**作为** 我系统中的最终用户 (体重管理 AI 助手)
**我想要** 一个专门的健康报告页面
**以便** 我可以直接查看最新的AI解读报告以及历史报告

---

## 🎯 验收标准

### 页面展示
- [ ] **默认视图**: 打开页面时自动显示当天的健康报告（最新生成的报告）  
- [ ] **AI解读渲染**: 正确显示AI生成的解读文本，格式美观、易于阅读
- [ ] **时间线浏览**: 用户应能浏览过去生成的历史报告
- [ ] **日期选择器**: 提供直观的日期/月份/年份选择器，用来查看历史报告

### 功能需求
- [ ] **报告筛选**: 按日期、报告类型（日报/周报/月报）进行过滤
- [ ] **分页展示**: 当历史报告过多时，实现分页展示每页最多20条
- [ ] **搜索功能**: 通过关键词搜索过去的报告内容
- [ ] **导出功能**: 将选定的报告导出为PDF或文本文件

### 性能要求
- [ ] **首次加载时间**: 页面应在3秒内完成首次加载
- [ ] **切换响应时间**: 在不同日期/类型间的页面切换响应时间应小于1秒
- [ ] **内存管理**: 对于大容量历史数据，实现适当的懒加载，避免一次性加载过多数据

---

## 📐 技术规格

### 文件位置

**前端文件:**
- `frontend/src/pages/HealthReports.jsx` - 偺康报告页面
- `frontend/src/components/ReportViewer.jsx` - 报告查看器组件  
- `frontend/src/components/ReportTimeline.jsx` - 报告时间线组件

**后端API:**
- 通过现有的`/reports`端点（已在Story 10.1 中实现）提供数据
- 可能需要添加额外的筛选和分页API支持

### 组件接口

**ReportViewer props:**
- `reportData: object` - 从API获取的报告数据
- `interpretation: string` - AI解释的自然语言文本
- `onReload: () => void` - 重新加载报告函数

**ReportTimeline props:**
- `reports: array` - 要显示的报告数组
- `onSelect: (reportId) => void` - 选择报告时回调
- `filters: object` - 可用的筛选选项

---

## 🛠️ 实施建议

### 数据流程
1. **初始化**: 加载默认的最新一份报告
2. **分页加载**: 在滚动到底部时加载更多报告，避免一次性获取所有报告数据
3. **前端缓存**: 本地缓存已加载的报告，提升切换速度

### UI 设计
1. 采用卡片式布局展示每份报告的主要信息
2. 为AI解读部分使用引述样式突出显示
3. 导航控件应放在清晰可见的位置
4. 日期选择器建议使用日历式UI

---

## 🚀 开发优先级

1. **基础页面构建**: 实现页面结构和默认报告展示
2. **历史浏览**: 添加时间线浏览和日期选择功能
3. **筛选搜索**: 实现报告筛选和全文搜索能力
4. **导出与性能**: 完善导出功能和性能优化
5. **UX优化**: 针对用户反馈调整UI细节和交互流畅性
