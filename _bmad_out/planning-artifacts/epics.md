# bmad - Epic Breakdown

## Overview

This document provides the complete epic and story breakdown for bmad, decomposing the requirements from the PRD, UX Design if it exists, and Architecture requirements into implementable stories.

## Requirements Inventory

### Functional Requirements

FR1: 用户注册与登录系统
- 用户通过邮箱注册账户
- 使用JWT进行身份验证
- 注册表单验证
- 重复邮箱检测
- 登录状态持久化

FR2: 用户档案管理
- 收集用户基本健康信息
- 必填字段：性别、出生日期、身高、当前体重、目标体重、活动水平
- 选填字段：饮食偏好、过敏食物、健康状况、减肥经历
- 自动计算并显示BMI
- 数据持久化到数据库

FR3: AI对话系统
- 基于QWen AI的对话系统
- 三种核心角色：营养师模式、行为教练模式、情感陪伴模式
- 动态角色切换
- 营养咨询、行为指导、情感支持

FR4: 营养管理
- 膳食计划生成
- 热量追踪
- 营养成分分析
- 个性化食谱推荐

FR5: 行为习惯养成
- 习惯打卡系统
- 进度追踪
- Streak记录
- 行为模式分析

FR6: 数据记录与分析
- 体重/体脂记录
- 运动数据同步
- 睡眠质量追踪
- 健康报告生成

FR7: 情感支持系统
- 情绪识别与回应
- 压力管理建议
- 积极心理暗示
- 危机时刻陪伴

FR8: 专业角色融合
- 三维评估系统
- 动态角色切换
- 跨领域分析

FR9: 激励与游戏化
- 积分系统
- 等级成长
- 徽章成就
- 挑战任务

FR10: 科学量化展示
- 数据可视化
- 趋势分析
- 证据支持建议
- 健康评分系统

FR11: 仪表盘功能
- 今日概览
- 快捷操作
- 进度图表
- AI建议卡片

FR12: 响应式设计
- 移动端适配（< 768px）
- 平板适配（768px - 1024px）
- 桌面适配（> 1024px）
- 触摸友好的按钮大小（最小44px）

FR13: 数据模型管理
- 用户表、用户档案表
- 对话表、消息表
- 营养计划表、饮食记录表
- 习惯表、习惯打卡表
- 健康数据表、游戏化数据表

### NonFunctional Requirements

NFR1: 可用性
- 新用户5分钟内上手核心功能
- 关键任务（记录体重、打卡）完成率 > 90%
- 用户操作错误率 < 5%
- 每个页面有上下文帮助

NFR2: 可靠性
- 系统可用性：99.5%（允许计划内维护）
- 数据持久性：100%（定期备份）
- 故障恢复：故障后 < 5分钟恢复
- 降级策略：AI服务不可用时，提供静态建议

NFR3: 可维护性
- 代码规范：遵循 PEP8（Python）和 Airbnb（JS/TS）规范
- 文档：核心代码必须有文档字符串
- 测试覆盖率：单元测试覆盖率 > 70%
- 日志：关键操作有日志记录

NFR4: 可扩展性
- 水平扩展：架构支持增加服务器实例
- 数据库：表结构设计支持未来扩展
- 模块化：功能模块解耦，便于新增功能

NFR5: 合规性
- 数据隐私：遵守中国《个人信息保护法》
- 医疗免责声明：AI建议仅供健康参考，非医疗诊断
- 用户协议：明确服务条款和隐私政策

安全考虑：

测试需求：

- Playwright (E2E测试)

### Story 1.1: 用户注册系统

作为新用户，
我想要通过邮箱注册账户，
以便我可以访问体重管理AI助手服务。

**Acceptance Criteria:**

**Given** 用户访问注册页面
**When** 用户输入有效的邮箱地址和符合要求的密码（最小8位，包含字母和数字）
**Then** 系统创建用户账户并发送验证邮件（MVP可跳过邮件验证）
**And** 系统生成JWT令牌用于后续认证

**Given** 用户尝试注册
**When** 用户输入已注册的邮箱地址
**Then** 系统显示错误消息"该邮箱已注册"
**And** 阻止账户创建

**Given** 用户尝试注册
**When** 用户输入不符合要求的密码（少于8位或只包含字母）
**Then** 系统显示密码要求提示
**And** 阻止表单提交

**Given** 数据库连接正常
**When** 用户成功注册
**Then** 系统在users表中创建新用户记录
**And** 密码使用bcrypt加密存储

**Given** 数据库连接失败
**When** 用户尝试注册
**Then** 系统显示友好的错误消息"系统暂时不可用，请稍后重试"
**And** 记录错误日志

**Given** 用户输入极端长度的数据
**When** 用户输入超过255字符的邮箱地址
**Then** 系统显示输入长度限制提示
**And** 阻止表单提交

**Given** 性能要求
**When** 用户提交注册表单
**Then** 注册请求响应时间 < 500ms
**And** 数据库插入操作时间 < 100ms

### Story 1.2: 用户登录系统

作为注册用户，
我想要使用邮箱和密码登录系统，
以便我可以访问我的个人健康数据和AI助手服务。

**Acceptance Criteria:**

**Given** 用户访问登录页面
**When** 用户输入正确的邮箱和密码
**Then** 系统验证用户凭证并生成JWT令牌
**And** 用户被重定向到仪表板页面

**Given** 用户访问登录页面
**When** 用户输入错误的密码
**Then** 系统显示错误消息"邮箱或密码不正确"
**And** 保持用户未登录状态

**Given** 用户访问登录页面
**When** 用户输入未注册的邮箱
**Then** 系统显示错误消息"邮箱未注册"
**And** 提供注册链接

**Given** 用户已登录
**When** 用户刷新页面或重新访问应用
**Then** 系统使用JWT令牌保持用户登录状态
**And** 用户无需重新登录

**Given** 数据库查询性能要求
**When** 用户登录
**Then** 用户凭证验证时间 < 200ms
**And** JWT令牌生成时间 < 50ms

**Given** 安全要求
**When** 用户连续5次输入错误密码
**Then** 系统暂时锁定该账户15分钟
**And** 发送安全警告邮件给用户

**Given** JWT令牌过期
**When** 用户使用过期的令牌访问受保护页面
**Then** 系统重定向用户到登录页面
**And** 显示"会话已过期，请重新登录"消息

**Given** 并发登录场景
**When** 多个用户同时登录
**Then** 系统能处理至少100个并发登录请求
**And** 平均响应时间保持在 < 500ms

作为已登录用户，

**And** 调用AI API获取回复

**Given** AI API返回回复

**Then** 系统显示AI回复在聊天历史中
**And** 保持对话历史可滚动查看

**Given** 网络连接问题

**And** 提供重试选项

**Given** 性能要求
**When** 用户发送消息给AI
**Then** AI首字响应时间 < 2秒
**And** 完整响应生成时间 < 10秒

**Given** 数据库要求
**When** 用户发送消息
**Then** 系统在messages表中保存用户消息
**And** 在conversations表中更新对话记录

**Given** 长消息处理
**When** 用户输入超过1000字符的消息
**Then** 系统提示消息长度限制
**And** 允许用户编辑或分段发送

**Given** 空消息处理
**When** 用户尝试发送空消息
**Then** 系统提示"消息不能为空"
