## ADR编号规则
- **ADR-001**: 第一个架构决策
- **ADR-002**: 第二个架构决策
- **ADR-XXX**: 按顺序递增

## 决策状态说明
- **提议**: 正在讨论中，尚未最终决定
- **已接受**: 团队已接受并同意实施
- **已弃用**: 决策不再适用，但历史记录保留
- **已取代**: 被新的决策取代

## 工作流程
1. **识别需求**: 发现需要架构决策的问题
2. **创建ADR**: 使用此模板创建新ADR
3. **讨论评审**: 相关方讨论和评审
4. **决策记录**: 记录最终决策和理由
5. **实施跟踪**: 跟踪实施进度
6. **定期回顾**: 定期回顾决策的有效性
