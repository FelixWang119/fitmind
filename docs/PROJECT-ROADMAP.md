## 📖 使用指南

### 何时更新

| 场景 | 更新内容 | 频率 |
|------|---------|------|
| Story 完成 | 更新状态、完成日期 | 每个 Story 完成后 |
| 优先级调整 | 更新优先级列 | 随时 |
| Sprint 结束 | 更新进度条、统计数据 | 每周 |
| 新增 Story | 添加到对应 Epic | 按需 |

### 如何更新

1. **Story 完成时：**
   ```markdown
   | Story | 名称 | 状态 | 完成日期 |
   |-------|------|------|---------|
   | 1.7 | 数据库模型 | ✅ 完成 | 2026-03-01 | ← 更新这里
   ```

2. **更新进度条：**
   ```
   项目完成度：72% ████████████████░░░░░░░░
                ↑ 计算方式：完成 Story 数 / 总 Story 数
   ```

3. **更新统计表：**
   - 修改 `完成：X/Y Story`
   - 更新百分比

### 快速命令

```bash
# 查看当前进度
grep -E "完成|进度" docs/PROJECT-ROADMAP.md

# 查找待开始的 Story
grep "⏳" docs/PROJECT-ROADMAP.md

# 查找已完成的 Story
grep "✅" docs/PROJECT-ROADMAP.md
```
