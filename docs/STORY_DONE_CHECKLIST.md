# Story 完成检查清单 (Definition of Done)

## 使用说明
- 每个 Story 在完成前必须检查所有适用项
- **标⭐的是集成检查项，最容易遗漏**
- 所有检查项通过后才能移动到"完成"列

---

## ✅ 完成标准（所有 Story）

### 功能实现
- [ ] 所有验收标准（Acceptance Criteria）已实现
- [ ] 功能代码已完成
- [ ] 代码已提交并推送到远程仓库
- [ ] 没有未解决的合并冲突

### 代码质量
- [ ] 代码通过 lint 检查（Black/ESLint）
- [ ] 代码格式化（Prettier）
- [ ] 没有 TypeScript/Python 类型错误
- [ ] 复杂代码有适当的注释

### 测试
- [ ] 单元测试已添加并运行通过
- [ ] 测试覆盖了主要场景和边界情况
- [ ] 没有故意跳过的测试（skip）
- [ ] ⭐ **关键功能有 E2E 测试或集成测试**

### 代码审查
- [ ] 代码审查已完成
- [ ] 所有审查意见已解决
- [ ] 审查人批准合并

---

## 🔌 前端 Story 额外检查项

### 组件开发
- [ ] 组件已正确导出
- [ ] Props 和 Events 类型定义完整
- [ ] 样式符合设计规范
- [ ] 响应式设计（移动端正常显示）

### ⭐ 路由和导航（关键！）
- [ ] **新页面已在 App.tsx 中添加路由**
- [ ] **路由路径正确配置**
- [ ] **导航跳转逻辑已实现（navigate, Link）**
- [ ] **从其他页面可以访问到新页面**
- [ ] **受保护路由有认证检查**

### ⭐ 用户流程（关键！）
- [ ] **从入口到完成的用户流程已手动验证**
- [ ] **页面跳转流畅，没有 404 错误**
- [ ] **加载状态和错误状态已处理**
- [ ] **表单验证正常工作**

### ⭐ API 集成（关键！）
- [ ] **前端 API 调用与后端端点匹配**
- [ ] **请求和响应数据结构正确**
- [ ] **认证 token 正确传递**
- [ ] **错误处理友好（用户能看懂）**

---

## 🔧 后端 Story 额外检查项

### API 开发
- [ ] API 端点已实现
- [ ] 请求/响应模型定义完整
- [ ] 状态码使用正确
- [ ] 错误处理完善

### ⭐ 路由配置（关键！）
- [ ] **新端点已添加到 router**
- [ ] **路由前缀配置正确**
- [ ] **API 文档可访问（Swagger/Redoc）**
- [ ] **认证和授权已配置**

### 数据库
- [ ] 数据模型已定义
- [ ] 数据库迁移脚本已准备
- [ ] 迁移已测试（本地环境）
- [ ] 数据验证规则完整

### ⭐ 前后端集成（关键！）
- [ ] **前端能成功调用新 API**
- [ ] **数据格式一致（单位、字段名）**
- [ ] **CORS 配置正确**
- [ ] **响应性能可接受**

## 🎯 关键用户流程检查（最容易遗漏！）

### 注册登录流程 ⭐ 必查
- [ ] 用户可以成功注册新账户
- [ ] 注册后跳转到预期页面（如 Onboarding）
- [ ] 用户可以登录
- [ ] 登录后跳转到 Dashboard
- [ ] 密码验证规则前后端一致

### 核心功能流程 ⭐ 必查
- [ ] 从 Dashboard 可以访问新功能
- [ ] 完成功能操作后能返回主流程
- [ ] 没有死胡同页面（无法继续操作）

### 错误场景 ⭐ 必查
- [ ] 网络错误有友好提示
- [ ] 404 页面已配置
- [ ] 未认证用户访问受限页面会跳转登录

- [ ] 本地开发环境运行正常
- [ ] 关键用户流程手动测试通过

## 🚫 完成检查常见遗漏项

### 高频遗漏 Top 5
1. ❌ **路由配置了但没添加到 App.tsx**
2. ❌ **API 端点实现了但没添加到 router**
3. ❌ **注册/登录后没有跳转到预期页面**
4. ❌ **前端字段名与后端不一致**
5. ❌ **新页面没有添加到导航菜单**

### 检查方法
```bash
# 前端路由检查
grep -r "onboarding\|Onboarding" frontend/src/App.tsx

# 后端路由检查
grep -r "/nutrition" backend/app/api/v1/api.py

# 导航菜单检查
grep -r "/onboarding" frontend/src/components/layout/
```

## 🎯 与代码审查的关系

| 检查项 | Story 完成检查 | 代码审查 |
|--------|---------------|----------|
| 功能实现 | ✅ 开发者自查 | ✅ 审查人验证 |
| 代码质量 | ✅ 开发者自查 | ✅ 审查人检查 |
| 测试 | ✅ 运行测试 | ✅ 审查测试覆盖 |
| ⭐ 路由配置 | ✅ 开发者必须检查 | ✅ 审查人重点检查 |
| ⭐ 用户流程 | ✅ 手动验证 | ✅ 审查人抽检验证 |
| 文档 | ✅ 开发者更新 | ✅ 审查人检查 |

**关键区别：**
- **Story 完成检查**：开发者在提交审查前的**自我验证**
- **代码审查**：审查人的**独立验证**
