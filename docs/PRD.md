# 体重管理 AI 助手 - 产品需求文档 (PRD)

**版本**: 1.0  
**日期**: 2026年2月  
**状态**: MVP 阶段  

---

## 1. 产品概述

### 1.1 产品愿景
打造一个**低成本、高专业度、强情感陪伴**的 AI 体重管理助手，通过融合营养师与行为教练的双重专业能力，为肥胖人群提供科学、可持续的减重方案，帮助他们建立长期健康生活习惯。

### 1.2 产品定位
| 维度 | 定位 |
|------|------|
| **目标用户** | 肥胖人群（BMI ≥ 28），尤其关注中低收入群体 |
| **核心价值** | 专业体重管理 + 行为改变支持 + 情感陪伴 |
| **差异化** | 营养师+行为教练双角色融合、科学量化、长期陪伴 |
| **商业模式** | 低成本订阅（远低于传统营养师咨询费用） |
| **竞品差异** | 比通用健康App更专业，比真人营养师更便宜、更持久 |

### 1.3 一句话价值主张
> "以一杯咖啡的价格，获得专业营养师+行为教练7×24小时的个性化陪伴。"

## 4. 详细功能规格

### 4.1 用户系统

#### 4.1.1 注册与登录

**功能描述**
用户通过邮箱注册账户，使用JWT进行身份验证。

**用户流程**
1. 用户打开应用，显示登录页
2. 新用户点击"注册"，填写邮箱、密码、基本信息
3. 系统发送验证邮件（MVP可跳过邮件验证）
4. 用户登录后进入引导流程

**数据要求**
- 邮箱（唯一，必填）
- 密码（最小8位，需包含字母和数字）
- 昵称（选填，默认用邮箱前缀）

**验收标准**
- [ ] 注册表单验证正常
- [ ] 重复邮箱提示错误
- [ ] JWT Token 正确生成和验证
- [ ] 登录状态持久化

#### 4.1.2 用户档案

**功能描述**
收集用户的基本健康信息，为AI提供个性化服务依据。

**收集信息**
| 字段 | 类型 | 必填 | 说明 |
|-----|------|-----|------|
| 性别 | 枚举 | 是 | 男/女/其他 |
| 出生日期 | 日期 | 是 | 计算年龄 |
| 身高 | 数字 | 是 | 单位：厘米 |
| 当前体重 | 数字 | 是 | 单位：公斤 |
| 目标体重 | 数字 | 是 | 单位：公斤 |
| 活动水平 | 枚举 | 是 | 久坐/轻度/中度/重度 |
| 饮食偏好 | 多选 | 否 | 素食/清真/无麸质等 |
| 过敏食物 | 文本 | 否 | 自由填写 |
| 健康状况 | 多选 | 否 | 糖尿病/高血压/心脏病等 |
| 减肥经历 | 文本 | 否 | 简单描述 |

**验收标准**
- [ ] 所有必填项验证
- [ ] 自动计算BMI并显示
- [ ] 数据持久化到数据库

### 4.2 AI 对话系统

#### 4.2.1 核心能力

**功能描述**
基于 QWen AI 的对话系统，提供营养咨询、行为指导、情感支持。

**角色设定**
系统具有三种核心角色，可根据对话内容动态切换：

1. **营养师模式**
   - 专业术语：BMI、BMR、TDEE、宏量营养素
   - 能力：膳食分析、营养建议、食谱推荐
   - 语气：专业、准确、循证

2. **行为教练模式**
   - 专业术语：习惯养成、行为改变、认知重构
   - 能力：习惯设计、障碍克服、动机激发
   - 语气：鼓励、引导、赋能

3. **陪伴者模式**
   - 专业术语：情绪支持、正念、自我关怀
   - 能力：情绪识别、压力管理、情感陪伴
   - 语气：温暖、同理、支持

**对话场景示例**

**场景1：日常咨询**
> 用户："我今天吃了汉堡，是不是减肥失败了？"
> 
> AI："偶尔吃一顿汉堡完全没问题！减肥不是非黑即白的。重要的是长期趋势，不是单次选择。我们来分析一下：一个汉堡大约500-600卡路里，只要今天其他两餐稍微清淡一点，或者增加30分钟散步，完全可以平衡。关键是不要因为这一餐就产生'破罐破摔'的想法，下一顿正常吃就好。你今天晚餐有什么计划吗？"

**场景2：情绪支持**
> 用户："我已经两周没掉秤了，好沮丧"
> 
> AI："我能感受到你的沮丧，这真的很正常。体重平台期是减肥过程中几乎每个人都会经历的阶段。让我看看你的数据... 其实你的体重曲线整体是在下降的，这两周虽然数字没变，但看之前的趋势，你已经在健康减重了。有时候身体需要时间来适应新的体重。我们一起看看这周的饮食记录，也许能发现一些可以优化的地方？"

**场景3：专业指导**
> 用户："我想知道我每天应该吃多少蛋白质"
> 
> AI："根据你的身体数据（身高175cm，体重85kg），你的基础代谢率(BMR)是..."

**技术要求**
- 使用 QWen API（或兼容 OpenAI 格式的其他模型）
- 实现流式响应（SSE）
- 支持上下文记忆（最近20轮对话）
- 响应时间：AI生成时间 < 10秒

**验收标准**
- [ ] 对话能正常进行
- [ ] 能识别用户意图并切换到合适角色
- [ ] 响应时间符合要求
- [ ] 对话历史正确保存和显示

### 4.3 营养管理模块

#### 4.3.1 个人营养计划

**功能描述**
根据用户档案自动生成个性化营养计划。

**计算逻辑**
1. **BMR（基础代谢率）**
   - 男性：BMR = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄(y) + 5
   - 女性：BMR = 10 × 体重(kg) + 6.25 × 身高(cm) - 5 × 年龄(y) - 161

2. **TDEE（每日总能量消耗）**
   - TDEE = BMR × 活动系数
   - 久坐：1.2 | 轻度活动：1.375 | 中度活动：1.55 | 重度活动：1.725

3. **目标热量**
   - 减重：TDEE - 500 kcal/天（每周减重约0.5kg）
   - 维持：TDEE
   - 增肌：TDEE + 300 kcal/天

4. **宏量营养素分配**
   - 蛋白质：1.6-2.2g/kg 体重（至少25%热量）
   - 脂肪：20-35% 热量
   - 碳水：剩余热量

**输出内容**
- 每日目标热量
- 三大营养素克数
- 餐次分配建议（早30%、午40%、晚30%）
- 一周食谱推荐（考虑用户饮食偏好）

#### 4.3.2 膳食记录

**功能描述**
用户可以记录每日饮食，系统自动分析营养成分。

**输入方式**
1. **文字描述**："中午吃了一碗牛肉面"
2. **拍照上传**：（Phase 2）AI 识别食物
3. **语音输入**：（Phase 2）语音转文字

**分析能力**
- 估算食物热量
- 分析营养成分（蛋白质、脂肪、碳水）
- 评估与目标的差距
- 提供改进建议

**界面要求**
- 日历视图查看历史记录
- 当日热量进度条
- 营养素圆环图

**验收标准**
- [ ] 能记录三餐和加餐
- [ ] 正确计算当日总热量
- [ ] 显示与目标的对比
- [ ] 支持修改和删除记录

### 4.4 行为习惯系统

#### 4.4.1 习惯管理

**功能描述**
用户可以创建、追踪和管理健康习惯。

**习惯类型**
1. **系统推荐习惯**（基于用户目标自动建议）
   - 每日饮水2000ml
   - 每日步行8000步
   - 每晚睡眠7-8小时
   - 每周运动3次

2. **自定义习惯**
   - 用户自行创建
   - 设置频率（每日/每周几次）
   - 设置提醒时间

**习惯追踪**
- 打卡按钮（一键打卡）
- 连续天数（Streak）统计
- 完成率统计（本周/本月）
- 习惯热力图

**激励机制**
- 连续7天：小鼓励提示
- 连续30天：徽章奖励
- 打破记录时：积极重构（"没关系，重新开始，你之前做到过！"）

#### 4.4.2 行为改变支持

**功能描述**
AI 主动帮助用户克服行为改变的障碍。

**场景识别**
- 连续3天未打卡 → 主动询问遇到的困难
- 习惯完成率 < 50% → 建议降低难度
- 用户表达挫败感 → 提供认知重构建议

**干预策略**
1. **障碍解决**
   - 时间不够 → 建议微习惯（如：每天1个俯卧撑）
   - 忘记打卡 → 设置提醒
   - 缺乏动力 → 关联深层价值（"你是为了家人的健康"）

2. **认知重构**
   - 全或无思维 → 强调渐进改善
   - 负面自我评价 → 引导自我关怀

**验收标准**
- [ ] 能创建和删除习惯
- [ ] 打卡功能正常
- [ ] 正确计算streak
- [ ] AI能识别困难并提供支持

### 4.5 健康数据系统

#### 4.5.1 数据记录

**支持的数据类型**

| 数据类型 | 频率 | 输入方式 | 单位 |
|---------|------|---------|------|
| 体重 | 每日/每周 | 手动输入 | kg |
| 体脂率 | 每周 | 手动输入 | % |
| 腰围 | 每月 | 手动输入 | cm |
| 血压 | 每周（如有需要） | 手动输入 | mmHg |
| 血糖 | 按需 | 手动输入 | mmol/L |
| 步数 | 每日 | 手动/同步 | 步 |
| 睡眠时长 | 每日 | 手动输入 | 小时 |
| 运动 | 每次 | 手动输入 | 分钟/类型 |

#### 4.5.2 数据可视化

**图表类型**
- 折线图：体重趋势、BMI变化
- 柱状图：周运动量对比
- 饼图：营养素分布

**数据分析**
- 周/月/季度趋势
- 平均减重速度
- 预测达成目标日期
- 健康评分（综合指标）

**验收标准**
- [ ] 支持所有数据类型记录
- [ ] 图表正确显示数据
- [ ] 趋势分析准确
- [ ] 数据可导出（CSV）

### 4.6 情感支持系统

#### 4.6.1 情绪识别

**识别来源**
- 用户明确表达："我今天心情不好"
- AI 识别语气：沮丧、焦虑、兴奋等
- 行为数据：连续未打卡、体重波动大

**情绪类型**
- 挫败（减肥平台期、体重反弹）
- 焦虑（担心效果、怕失败）
- 孤独（缺乏支持）
- 压力（工作、生活）
- 自我怀疑
- 兴奋（取得进展）
- 感恩
- 疲惫
- 困惑

#### 4.6.2 支持策略

**策略库**

| 情绪 | 策略 | 示例回应 |
|-----|------|---------|
| 挫败 | 正常化 + 数据支持 | "平台期很正常，看整体趋势你已经在进步了" |
| 焦虑 | 降低要求 + 小目标 | "我们这周只关注一件事：多喝水" |
| 孤独 | 陪伴 + 社群建议 | "我在这里陪着你，你也可以加入我们的用户群" |
| 压力 | 减压建议 + 暂时减负 | "这周如果太忙，先保持现状就好，不用追求进步" |
| 自我怀疑 | 肯定过往努力 + 重构 | "你已经坚持了一个月，这就是能力的证明" |
| 兴奋 | 庆祝 + 防止冒进 | "太棒了！不过记得保持稳定节奏，不要一下子太激进" |

**危机识别**
- 关键词监测："不想活了"、"想放弃一切"
- 触发专业帮助建议（心理健康热线）

#### 4.6.3 陪伴机制

**主动关怀**
- 每天早上：天气 + 健康提醒
- 连续2天未登录："两天没见到你了，一切还好吗？"
- 达成里程碑：庆祝消息
- 生日/注册纪念日：特别祝福

**个性化调整**
- 根据用户反馈调整关怀频率（有人觉得烦，有人喜欢）
- 学习用户偏好（什么类型的鼓励最有效）

**验收标准**
- [ ] 能识别主要情绪类型
- [ ] 提供合适的支持回应
- [ ] 不过度打扰用户
- [ ] 能识别危机情况并引导

### 4.7 专业角色融合系统

#### 4.7.1 三维评估系统

**评估维度**

1. **营养维度**
   - 饮食结构合理性
   - 热量缺口适当性
   - 营养素充足性
   - 饮食行为（规律进餐、情绪性进食等）

2. **行为维度**
   - 习惯养成情况
   - 自我效能感
   - 障碍应对能力
   - 动机水平

3. **心理维度**
   - 情绪稳定性
   - 压力水平
   - 自我认知
   - 社会支持

**评估方法**
- 定期问卷（每周/每月）
- AI对话中的自然评估
- 行为数据自动分析

#### 4.7.2 角色切换逻辑

**触发条件**
- 用户主动询问营养问题 → 营养师模式
- 用户表达执行困难 → 行为教练模式
- 用户情绪波动 → 陪伴者模式
- 复杂问题 → 混合模式（先分析营养，再提供行为建议，最后情感支持）

**切换提示**
- 不明显切换，自然过渡
- 可以明确说明："作为你的营养师，我建议..."、"作为你的教练，我想帮你..."

#### 4.7.3 跨领域分析示例

**用户问题**: "我总是晚上忍不住吃零食"

**营养师角度**: "晚上的零食可能让你一天的热量超标。我们可以调整晚餐结构，增加蛋白质和纤维，让你更有饱腹感。"

**行为教练角度**: "这是典型的'夜间进食'模式。通常不是因为饿，而是因为习惯或情绪。我们可以尝试：1）晚饭后刷牙作为'停止信号'；2）识别触发因素（是无聊？压力？）；3）用其他行为替代（喝茶、散步）。"

**陪伴者角度**: "改变习惯很难，不要责怪自己。很多人都有这个挑战。你已经意识到这个问题，这就是改变的第一步。我们一起慢慢来。"

**综合回应**:
> "晚上想吃零食是很常见的挑战。从营养角度，我们可以调整晚餐，让它更饱腹。但更重要的是理解为什么会这样——通常不是因为饿，而是习惯或情绪。你晚上一般是什么情况下想吃零食呢？是无聊、压力大，还是已经成了习惯？我们可以一起设计一个应对策略，比如晚饭后立即刷牙，或者准备些低热量的替代品。不用责怪自己，很多人都有这个挑战，重要的是找到适合你的方法。"
