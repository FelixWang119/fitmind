# 代码审查清单 (Code Review Checklist)

## 使用说明
- 每个 Story 的代码审查必须覆盖相关检查项
- 标记 ✅ (通过) / ⚠️ (需要注意) / ❌ (未实现)
- 所有 P0 项必须通过才能合并代码

---

## 📋 通用检查项（所有 Story）

### 代码质量
- [ ] 代码遵循项目的代码风格规范（Black/ESLint/Prettier）
- [ ] 没有明显的代码重复（DRY 原则）
- [ ] 变量和函数命名清晰易懂
- [ ] 复杂逻辑有适当的注释
- [ ] 没有调试代码（console.log, breakpoint 等）

### 安全性
- [ ] 用户输入已验证和清理
- [ ] 没有硬编码的敏感信息（密码、密钥等）
- [ ] API 端点有适当的认证和授权
- [ ] SQL 查询使用参数化（防止 SQL 注入）

### 测试
- [ ] 单元测试已添加并运行通过
- [ ] 测试覆盖了主要功能和边界情况
- [ ] 没有跳过关键测试（无 @skip 标记）

---

## 🔌 前端特定检查项

### 组件开发
- [ ] 组件职责单一，功能内聚
- [ ] Props 类型定义完整（TypeScript）
- [ ] 状态管理合理（避免过度使用 useState）
- [ ] 副作用处理正确（useEffect 清理）
- [ ] 错误处理完善（Error Boundary）

### 路由和导航 ⭐ 关键
- [ ] **新页面已在 App.tsx 中配置路由**
- [ ] **路由路径符合项目命名规范**
- [ ] **导航跳转逻辑正确（navigate, Link）**
- [ ] **路由参数处理正确（:id, query params）**
- [ ] **受保护路由有认证检查**

### 用户体验
- [ ] 加载状态已处理（Loading）
- [ ] 错误状态有友好提示
- [ ] 空状态有适当的引导
- [ ] 响应式设计（移动端适配）
- [ ] 可访问性（ARIA 标签）

### 前端集成 ⭐ 关键
- [ ] **API 调用已正确配置（axios, fetch）**
- [ ] **环境变量使用正确（import.meta.env）**
- [ ] **组件导入路径正确（@/ 别名）**
- [ ] **跨组件通信正常（props, events, store）**

---

## 🔧 后端特定检查项

### API 设计
- [ ] 端点路径符合 REST 规范
- [ ] 请求和响应模型定义完整（Pydantic）
- [ ] 状态码使用正确（200, 201, 400, 404, 500 等）
- [ ] 错误响应格式统一

### 数据库
- [ ] 数据库模型定义正确（SQLAlchemy）
- [ ] 查询性能优化（索引、懒加载）
- [ ] 事务处理正确（commit, rollback）
- [ ] 数据迁移脚本已准备（Alembic）

### 业务逻辑
- [ ] 服务层职责清晰
- [ ] 异常处理完善
- [ ] 日志记录适当（不包含敏感信息）
- [ ] 配置项从环境变量读取

---

## 🔄 集成检查项 ⭐ 关键（容易遗漏）

### 前后端集成
- [ ] **前端 API 调用与后端端点匹配**
- [ ] **请求和响应数据结构一致**
- [ ] **认证流程端到端可用**
- [ ] **CORS 配置正确**

### 路由集成 ⭐ 关键
- [ ] **用户从 A 页面到 B 页面的导航流程已验证**
- [ ] **登录/注册后跳转逻辑正确**
- [ ] **404 页面已配置**
- [ ] **路由守卫逻辑正确（权限控制）**

### 关键用户流程 ⭐ 关键
- [ ] **注册 → 登录 → 主页 流程已验证**
- [ ] **新功能的入口可访问**
- [ ] **完成功能后能返回主流程**

---

## 📝 Story 特定检查项

### 根据 Story 类型选择检查

#### 新功能 Story
- [ ] 功能完整实现所有验收标准
- [ ] 用户文档/更新日志已更新
- [ ] 功能开关已配置（如适用）

#### Bug 修复 Story
- [ ] 根本原因已分析
- [ ] 修复已验证，问题不再现
- [ ] 添加了防止回归的测试

#### 技术改进 Story
- [ ] 性能提升已测量
- [ ] 技术债务已记录
- [ ] 对现有功能无负面影响

---

## ✅ 审查完成标准

### 必须满足（P0）
- [ ] 所有 P0 检查项通过
- [ ] 没有严重安全问题
- [ ] 关键用户流程可正常工作
- [ ] 单元测试运行通过

### 建议满足（P1/P2）
- [ ] 代码质量检查通过
- [ ] 文档完整
- [ ] 性能影响可接受

---

## 🚫 常见问题检查

### 前端高频问题
- ❌ 组件导入路径错误（相对路径 vs 别名路径）
- ❌ 路由配置了但忘记添加到 App.tsx
- ❌ API 调用使用了错误的 baseURL
- ❌ 环境变量未定义或使用错误
- ❌ 表单提交没有处理 loading 状态

### 后端高频问题
- ❌ 数据库字段单位不一致（克 vs 千克）
- ❌ 验证规则前后端不一致
- ❌ 忘记添加新的路由到 router
- ❌ 异常没有正确抛出和捕获
- ❌ 日志包含敏感信息

### 集成高频问题
- ❌ 前端字段名与后端不一致（camelCase vs snake_case）
- ❌ 认证 token 没有正确传递
- ❌ 跨域请求未配置 CORS
- ❌ 文件上传缺少 multipart/form-data
