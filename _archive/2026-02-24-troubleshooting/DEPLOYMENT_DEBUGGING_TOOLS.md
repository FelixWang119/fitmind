# 前后端服务调试工具

本目录包含一组用于管理和调试前后端服务的实用脚本。

## 当前服务配置

- **后端服务 (FastAPI)**: 端口 8000
- **前端服务 (React/Vite)**: 端口 3000

## 所有脚本功能

### 1. check_status.sh
检查当前正在运行的服务状态。

```bash
./check_status.sh
```

### 2. debug_backend.sh
专门管理后端服务 (端口 8000) 的调试器启动脚本。
- 检查端口 8000 是否已有进程运行
- 如果有运行中的进程，则停止它们
- 启动新的后端服务实例
- 进行健康检查确保服务正常

```bash
./debug_backend.sh
```

### 3. debug_backend_8001.sh
专门管理后端服务 (端口 8001) 的调试器启动脚本 - 为兼容原配置创建。
- 检查端口 8001 是否已有进程运行
- 如果有运行中的进程，则停止它们
- 启动新的后端服务实例
- 进行健康检查确保服务正常

```bash
./debug_backend_8001.sh
```

### 4. debug_frontend.sh
专门管理前端服务 (端口 3000) 的调试器启动脚本。
- 检查端口 3000 是否已有进程运行
- 如果有运行中的进程，则停止它们
- 启动新的前端服务实例

```bash
./debug_frontend.sh
```

### 5. debug_both.sh
同时管理前后端服务的脚本，依次执行上述两个脚本并提供汇总结果。

```bash
./debug_both.sh
```

## 重要配置说明

问题解决: 前端的 `.env` 文件中的 `VITE_API_URL` 已经从 `http://localhost:8001/api/v1` 更新为 `http://localhost:8000/api/v1`，以兼容实际运行的后端服务端口。

如果需要恢复原始配置，可以在 `/Users/felix/bmad/frontend/.env` 中修改 `VITE_API_URL` 变量。

## 高级说明

- 所有脚本都会自动处理端口占用问题
- 在杀进程后等待直到进程完全退出，避免僵尸进程
- 在启动新服务后会进行检查，确保服务正常工作
- 脚本适用于开发环境中快速重启服务的场景

注意：根据当前检查，您的服务（后端在8000端口，前端在3000端口）应该都已能在正确的端口上通信。
